# AquaTrack Docs CMS - Visual Improvements

Aesthetic comparison between the Docusaurus site (localhost:3002) and the Astro + Keystatic CMS site (localhost:4321), with a prioritized list of fixes.

---

## What the Astro CMS site does well

- Dark theme is cohesive and looks modern
- Status/category badges with color coding (green/yellow/red/purple) are more informative than Docusaurus
- Card-grid layout on listing pages is visually richer than Docusaurus's plain tables
- Phase-grouped sidebar on Roads is excellent
- Category-grouped sidebar on ADRs is useful
- Home dashboard with item counts per section is a great CMS overview

---

## P0 - Critical (Broken / unusable) ✅

### ~~1. Markdown content not rendering on detail pages~~ ✅

Added `marked` dependency. Created `renderMarkdown()` helper in `lib/reader.ts`. Updated all 11 `[slug].astro` detail pages to convert raw markdown to HTML before injecting via `set:html`.

### ~~2. Date formatting is broken~~ ✅

Added `formatDate()` helper in `lib/reader.ts`. Updated all detail pages and the ADR index listing to format dates as `YYYY-MM-DD`.

### ~~3. Missing favicon~~ ✅

Created `public/favicon.svg` (water droplet icon). Added `<link rel="icon">` to `BaseLayout.astro`.

---

## P1 - High (Significant visual gaps vs Docusaurus) ✅

### ~~4. No footer~~ ✅

Added 4-column footer to `BaseLayout.astro` with Domain, BDD Testing, Planning, and Community sections. Matches Docusaurus structure with proper dark-theme styling, responsive 2-column on mobile.

### ~~5. No breadcrumb navigation~~ ✅

Added breadcrumb navigation to `DocLayout.astro`. All 11 detail pages now pass breadcrumbs (`Home › Collection › Item`). Styled with muted separators and hover states.

### ~~6. No previous/next navigation~~ ✅

Added previous/next link cards at the bottom of every detail page. Added `getPrevNext()` helper in `reader.ts`. All 11 detail pages pass prev/next props to DocLayout.

### ~~7. No table of contents (TOC) on detail pages~~ ✅

Added right-side TOC sidebar to `DocLayout.astro`. Extracts h2/h3 headings from rendered HTML. Heading IDs auto-generated by custom `marked` renderer. Supports h2/h3 indentation levels. Hides on narrow viewports (<1100px).

### ~~8. No code block syntax highlighting~~ ✅

Added `highlight.js` and `marked-highlight` packages. Configured `marked` with `markedHighlight` extension for server-side syntax highlighting. Using `github-dark` theme CSS. All code blocks across all 159+ pages now get language-aware highlighting.

### ~~9. No "Edit this page" link~~ ✅

Added per-page "Edit this page" link to `DocLayout.astro` with pencil icon SVG. All 11 detail pages pass `editUrl` pointing to `https://github.com/anomalyco/aquatrack/edit/main/docs/{collection}/{slug}.md`.

---

## P2 - Medium (Polish & refinement) ✅

### ~~10. No logo/icon~~ ✅

Inlined the water droplet SVG from `favicon.svg` into the header brand link in `BaseLayout.astro`. Displays at 22x22px next to "AquaTrack Docs" text.

### ~~11. No light/dark mode toggle~~ ✅

Added full light/dark theme support. Light mode CSS variables defined under `html[data-theme="light"]`. Toggle button in header with sun/moon icons. Respects `prefers-color-scheme` on first visit. Persists choice in `localStorage`. Swaps `highlight.js` stylesheet between `github-dark` and `github`. Badge colors adjusted for light mode.

### ~~12. No search~~ ✅

Added client-side search overlay with Cmd+K (Ctrl+K) keyboard shortcut. Search index built at render time from all 11 collections. Fuzzy matching on title and category. Keyboard navigation (arrows + Enter). Click-outside and Esc to close. Styled as a modal with result list and footer hints.

### ~~13. Home hero description is too generic~~ ✅

Updated hero to say "Municipal Water Infrastructure Management System" and added 3 action buttons: "Explore DDD" (primary), "View Roadmap" (secondary), "BDD Workflow" (secondary). Each has an inline SVG icon.

### ~~14. Docusaurus home has feature cards with icons~~ ✅

Added 3 feature cards between the hero and section grid: Water Usage Tracking (blue), DDD Architecture (purple), BDD-Driven Development (green). Each has a colored icon container with custom SVG, heading, and description paragraph.

### ~~15. Sidebar doesn't highlight current page~~ ✅

Fixed trailing-slash mismatch in `DocLayout.astro` — `currentPath` now strips trailing slashes before comparison. Active state (blue text, blue left border, subtle background) confirmed working.

### ~~16. ADR listing: sidebar not grouped on detail pages~~ ✅

ADR detail page sidebar now matches the index page — grouped by Architecture, Infrastructure, Security, Performance categories with an "Other" catch-all. Each category is its own collapsible section.

### ~~17. No collapsible sidebar sections~~ ✅

All sidebar sections are now collapsible via click. Title rendered as `<button>` with chevron SVG that rotates on collapse. Auto-collapses sections with >10 items that don't contain the active link (keeps Navigation always open). Uses `aria-expanded` for accessibility.

---

## P3 - Low (Nice-to-have) ✅

### ~~18. No responsive mobile menu~~ ✅

Added hamburger button in header (visible only on mobile ≤768px). Toggles sidebar as a fixed overlay with semi-transparent backdrop. Hides nav links on mobile to save space.

### ~~19. The Astro dev toolbar is visible~~ ✅

Added CSS rule `astro-dev-toolbar { z-index: 1000 !important; }` to ensure the dev toolbar renders above content without overlapping.

### ~~20. No scroll-to-top behavior~~ ✅

Added a scroll-to-top button (fixed, bottom-right) that appears after scrolling 400px. Smooth-scrolls to top on click. Styled with hover state matching the theme.

### ~~21. No link hover states on sidebar~~ ✅

Added `border-left-color: var(--color-text-muted)` on sidebar link hover, providing a subtle left-border indicator like Docusaurus.

### ~~22. Card hover effects could be stronger~~ ✅

Updated card hover to `translateY(-3px)` with `box-shadow: 0 8px 24px rgba(0,0,0,0.2)`. Home section cards also enhanced with `translateY(-2px)` and shadow. Transitions include `box-shadow` for smooth animation.

---

## Summary

All 22 items completed across P0 (Critical), P1 (High), P2 (Medium), and P3 (Low) priorities. The Astro + Keystatic CMS site now matches or exceeds the Docusaurus site in aesthetics and functionality.
